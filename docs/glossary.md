# Глоссарий

- **exteraGram** — форк Telegram с поддержкой Python-плагинов.
- **Chaquopy** — среда исполнения Python (3.11) внутри Android-приложения,
  встроенная в exteraGram; через неё работают плагины.
- **SDK** — поставляемые exteraGram скомпилированные модули (`base_plugin`,
  `android_utils`, `client_utils`, `file_utils`, `hook_utils`, `ui.*`,
  `extera_utils.*` и др.), доступные плагину по импорту.
- **Плагин** — один файл `<plugin_id>.py`, содержащий подкласс `BasePlugin`.
- **plugin_id** — уникальный идентификатор плагина; равен имени файла без
  расширения; задаётся строковым литералом (требование AST-парсера движка).
- **Дандеры метаданных** — модульные переменные `__id__`, `__name__`,
  `__version__`, `__description__`, `__author__`, `__icon__`,
  `__min_version__`/`__app_version__`, `__sdk_version__`, `__requirements__`.
  Читаются движком статически через AST, поэтому только строковые литералы.
- **PythonPluginsEngine** — компонент exteraGram, загружающий и выгружающий
  плагины.
- **Bundler** — ядро catalib, собирающее дерево исходников в один файл.
- **Встраиваемый загрузчик** — компактный код на `sys.meta_path`, помещаемый
  в начало выходного файла; регистрирует встроенные модули плагина.
- **Карта строк (source map)** — соответствие позиций во встроенном исходнике
  исходным файлам и строкам для корректных трейсбеков через `linecache`.
- **Манифест** — `catalib.toml`, описывающий метаданные и параметры сборки.
- **Среда инструмента** — исполнение catalib на машине разработчика.
- **Встраиваемая среда** — исполнение `runtime`/`support` внутри плагина под
  Chaquopy на устройстве.
- **Граница сред** — запрет встраиваемому коду зависеть от среды инструмента и
  внешних пакетов; проверяется тестом.
- **Зонд** — read-only плагин для эмпирической проверки поведения Chaquopy.
- **Dev server** — встроенный в exteraGram сервер разработки на TCP 42690;
  принимает JSON-команды (`write_plugin`, `reload_plugin`,
  `set_plugin_enabled`, `delete_plugin`). Штатный канал доставки плагина на
  устройство без root; используется модулем `deploy`.
