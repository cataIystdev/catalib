# Обзор системы

catalib — offline-инструмент, превращающий модульное дерево исходников плагина
exteraGram в один самодостаточный файл `<plugin_id>.py`, пригодный для штатной
установки и распространения.

## Контекст exteraGram

Плагины исполняются под Chaquopy (Python 3.11), встроенным в приложение. SDK
поставляется скомпилированными модулями. Движок (`PythonPluginsEngine`) загружает
плагин так: устанавливает pip-зависимости, обновляет кеши импорта, затем
`importlib.import_module("<plugin_id>")`, находит подкласс `BasePlugin`,
инстанцирует его и вызывает `on_plugin_load()`. Плагином считается только файл
верхнего уровня `*.py`; соседние файлы движком не управляются. Установка и
распространение для конечного пользователя — однофайловые.

## Две среды исполнения

- **Среда инструмента** — CPython 3.11+ на машине разработчика. Слои `cli`,
  `manifest`, `bundler`, `deploy`, `scaffold`.
- **Встраиваемая среда** — Chaquopy 3.11 на устройстве; исходники слоёв
  `runtime` и `support` попадают внутрь собранного плагина. Этот код зависит
  только от стандартной библиотеки и SDK exteraGram.

Граница между средами жёсткая и проверяется тестом границы.

## Поток сборки

1. `manifest` читает и валидирует `catalib.toml`.
2. `bundler.discovery` строит дерево модулей из каталога `src`.
3. `bundler.requirements` сливает `__requirements__`.
4. `bundler.sourcemap` строит карту строк для трейсбеков.
5. `bundler.compiler` формирует выходной файл: литералы метаданных,
   встроенный загрузчик из `runtime`, встроенные исходники плагина и
   вендоренные модули `catalib.support` (`bundler.vendor`), экспорт точки входа.
6. `manifest.metadata` статически (AST) проверяет корректность метаданных в выходе.
7. `deploy` (опционально) доставляет файл на устройство через dev server
   exteraGram (TCP 42690: `write_plugin` + `reload_plugin`) и перезагружает
   плагин. Прямой `adb push` в приватный каталог без root запрещён (ADR-0004).

## Поток на устройстве

Движок импортирует `<plugin_id>`. Встроенный загрузчик регистрируется в
`sys.meta_path` и лениво исполняет встроенные исходники под уникальным приватным
префиксом, изолируя плагин от других. Подкласс `BasePlugin` доступен на верхнем
уровне модуля; `support` предоставляет безопасные обёртки SDK.
