# ADR-0006: Паритет support-слоя с публичным SDK exteraGram

- **Статус:** принято
- **Дата:** 2026-05-18

## Контекст

Слой `catalib.support` (ADR-0003) покрывал лишь часть публичного SDK
exteraGram. Чего не хватало относительно документации
(<https://plugins.exteragram.app/docs>) и SDK:

- настройки: только `header`/`switch`/`input`/`text`; отсутствовали
  `Divider`, `Selector`, `EditText`, `Custom` и параметры `on_click`,
  `on_change`, `icon`, `accent`, `red`, `link_alias`, `create_sub_fragment`,
  `multiline`, `max_length`, `mask`. Кликабельную строку настроек плагины
  собирали вручную, перебирая API клика (наблюдалось в плагине backrooms);
- пункт меню: не пробрасывались `item_id`, `condition`, `priority`;
- не было декларативной обработки `on_app_event` (`AppEvent`);
- не было поддержки Xposed-хуков (`find_class`, `MethodHook`,
  `MethodReplacement`, `HookFilter`, `hook_filters`) — их приходилось
  импортировать из SDK напрямую, теряя офлайн-тестируемость.

Жёсткое требование: **не сломать ничего**. Слой `support` вендорится в
собранные плагины (например backrooms уже содержит его копию). Любое
изменение публичной поверхности обязано быть обратносовместимым.

## Решение

Довести `support` до паритета с публичным SDK, строго аддитивно:

- **Настройки.** Добавлены компоненты `divider`, `selector`, `edit_text`,
  `custom`; `switch`/`text_input`/`text` расширены keyword-параметрами SDK.
  Кликабельная строка стала первоклассной: `settings.text(..., on_click=…)`.
  Принцип неизменности: прежние позиционные сигнатуры сохранены, новые
  параметры — только keyword и со «незаданным» значением по умолчанию,
  поэтому для прежнего кода итоговый `params` (и вызов `ui.settings`) не
  меняется. Новая валидация добавлена только в новые функции
  (`selector`/`custom`), у прежних — без новых проверок.
- **Меню.** `menu_item`/`MenuSpec` получили keyword-поля `item_id`,
  `condition`, `priority`; пробрасываются в `MenuItemData` только когда
  заданы (прежний вызов формирует тот же объект).
- **События приложения.** Декоратор `@hook.app_event(*events)` и диспетчер
  `CatalibPlugin.on_app_event`. Прямое переопределение `on_app_event`
  подклассом сохранено; пустой плагин по-прежнему без поведения.
- **Xposed.** Модуль `support/xposed.py`: декоратор `@xposed(...)` с
  авто-регистрацией в `on_plugin_load` и авто-`unhook` в
  `on_plugin_unload`; мост `MethodHook`/`MethodReplacement`; проброс
  `HookFilter` через `hook_filters`. Ошибки рефлексии логируются и не
  роняют загрузку (pitfall #7). Прежнее поведение `on_plugin_unload` (его
  отсутствие у плагинов без Xposed) сохранено; добавлен переопределяемый
  `on_unload`.
- **Безопасные импорты.** `support.sdk` независимыми `try/except` импортирует
  и заглушает `AppEvent`, `MethodHook`, `MethodReplacement`, `BaseHook`,
  `HookFilter`, `hook_filters`, `find_class`. Обособлены от ядра: отсутствие
  имени на старой сборке SDK не сбрасывает `SDK_AVAILABLE`.
- Новый модуль `xposed.py` добавлен в вендоринг (`bundler.vendor`), иначе
  собранные плагины не загрузились бы; инвариант закреплён интеграционным
  тестом сборки.

Обратная совместимость публичной поверхности закреплена тестом: набор
имён до 0.2.0 обязан сохраниться, пересборка backrooms и его тесты —
зелёные.

## Альтернативы

- **Тонкие ре-экспорты SDK без декларативных слоёв.** Отвергнуто частично:
  ре-экспорты сделаны (для офлайн-тестируемости), но без `@xposed`/
  `@hook.app_event` осталась бы ровно та проблема забытой регистрации и
  ручного шаблона, ради устранения которой существует ADR-0003.
- **Скрыть SDK за фасадом, не давать прямого доступа.** Отвергнуто:
  противоречит ADR-0003 (слой не скрывает SDK, лишь убирает шаблон).
- **Менять прежние сигнатуры под «красивый» единый стиль.** Отвергнуто:
  ломает вендоренные копии в сторонних плагинах (нарушение требования).

## Последствия

- Плагины пишут полный спектр настроек и Xposed-хуки декларативно и
  офлайн-тестируемо; кликабельная строка не требует ручного перебора API
  (backrooms переведён на `settings.text(on_click=…)`).
- Публичная поверхность только расширяется; вендоренные копии в старых
  плагинах продолжают работать.
- `support` вырос; `xposed` вынесен в отдельный модуль (SRP, размер файла).
