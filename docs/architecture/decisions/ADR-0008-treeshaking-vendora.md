# ADR-0008: Помодульный отбор вендоренного catalib (tree-shaking)

- **Статус:** принято, подтверждено на реальных плагинах
- **Дата:** 2026-05-18

## Контекст

С ростом паритета (ADR-0006, ADR-0007) `catalib.support` стал большим
(~3.2 тыс. строк, 14 модулей-областей). `bundler/vendor.py` вшивал **все**
модули безусловно, а `catalib/support/__init__.py` **эагерно импортирует
все подмодули**. Поэтому каждый собранный плагин нёс весь SDK-паритет,
даже если использовал только `CatalibPlugin` и пару хуков — собранные
файлы получались тяжёлыми.

Требование пользователя: вшивать только то, что плагин реально использует
из `catalib` (не просто файлы плагина, а именно нужные модули `catalib`).

## Решение

Добавлен этап `bundler/treeshake.py`, выполняемый компилятором:

1. Статически (AST, `ast.walk` — включая импорты внутри функций) собрать,
   что плагин импортирует из `catalib`: символы `from catalib.support
   import X`, подмодули `from catalib.support.sub ...` /
   `import catalib.support.sub`.
2. Из настоящего `catalib/support/__init__.py` построить карту
   «символ → подмодуль» и множество ре-экспортируемых подмодулей.
3. Построить граф зависимостей внутри `catalib` и взять транзитивное
   замыкание от используемых плагином модулей. Узел пакета
   `catalib.support` в графе не наследует зависимости полного `__init__`
   (он импортирует всё) — вместо него генерируется **урезанный
   `__init__`**, импортирующий только нужные подмодули/символы.
4. В `_CATALIB_SOURCES` попадают только модули замыкания; для
   `catalib.support` — сгенерированный урезанный `__init__`.

Гранулярность — **помодульная** (символьное вырезание не делается:
рефлексия/class proxy/динамика SDK делают его небезопасным).

Режим по умолчанию — `auto`. Консервативность: при неоднозначных
импортах (`import catalib.support`/`from catalib import support` как
объект, `from catalib.support import *`, несопоставимое имя, импорт
`catalib` вне `catalib.support`, синтаксическая ошибка) выполняется
**полный вендоринг с предупреждением** — лучше больше байт, чем сломанный
плагин. Опт-аут — `[build] vendor = "full"`.

## Альтернативы

- **Символьный tree-shaking (вырезать неиспользуемые функции/классы).**
  Отвергнуто: хрупко при рефлексии, class proxy и динамическом доступе к
  SDK; риск молча сломать плагин ради небольшого выигрыша.
- **Ленивый `__init__` (PEP 562 `__getattr__`) в самом пакете.**
  Отвергнуто как основной механизм: меняет рантайм-семантику
  установленного пакета и не убирает исходники отсечённых модулей из
  бандла. Генерация урезанного `__init__` только для сборки проще и
  точнее, не затрагивая dev/тесты.
- **Прунить и модули самого плагина.** Отложено: пользователь явно
  ограничил охват `catalib`; модули плагина могут грузиться динамически.

## Последствия

- Существенно меньший собранный файл (на `example` и backrooms объём
  `catalib`-части снизился ≈ на треть; отсечено 9 из 16 модулей).
- Поведение обратносовместимо: при сомнении — полный вендоринг.
- Появился публичный контракт манифеста `[build] vendor`.
- Корректность закреплена: юнит-тесты `treeshake`, тест манифеста,
  тест компилятора и существующие интеграционные тесты исполнения
  урезанного бандла в чистом интерпретаторе (host-`catalib` заблокирован);
  дополнительно проверено ручной сборкой стороннего плагина backrooms.
