# ADR-0002: Сборка в один файл со встраиваемым загрузчиком на sys.meta_path

- **Статус:** принято (эмпирически подтверждается зондом T-301)
- **Дата:** 2026-05-17

## Контекст

Декомпиляция загрузчика exteraGram (`PythonPluginsEngine.loadPlugin`) показала:

1. Плагин — ровно один файл `<plugin_id>.py` в каталоге плагинов; имя модуля
   строго равно `<plugin_id>`.
2. Загрузка: `refreshImportCaches(pluginId, каталог)` → `python.getModule(pluginId)`
   (Chaquopy выполняет `importlib.import_module`).
3. Перечисляются только файлы верхнего уровня, оканчивающиеся на `.py`;
   подкаталоги плагинами не считаются.
4. При выгрузке из `sys.modules` удаляется только ключ `<plugin_id>`; соседние
   файлы движком не управляются и не очищаются.
5. Установка и распространение для конечного пользователя — однофайловые.

Следовательно, многофайловая разработка при сохранении штатной модели установки
требует сборки в один файл. Способ внутреннего связывания модулей внутри этого
файла — предмет данного решения.

## Решение

Bundler встраивает исходный текст каждого модуля плагина в выходной `.py` как
данные. В начало файла помещается компактный загрузчик: `MetaPathFinder` и
`Loader` поверх `importlib`, который при первом импорте лениво исполняет
встроенный исходник нужного модуля. Все встроенные модули регистрируются под
уникальным приватным префиксом, производным от `plugin_id`, чтобы исключить
коллизии в общем `sys.modules`. Подкласс `BasePlugin` экспортируется в
пространство имён выходного модуля, чтобы движок нашёл точку входа штатно.
Исходники регистрируются в `linecache` под исходными путями — трейсбеки
указывают на реальные файлы разработчика.

Применимость кастомного `sys.meta_path` в среде Chaquopy проверяется зондом
T-301 на реальном устройстве **до** реализации ядра. Результат фиксируется
дополнением к этому ADR.

## Альтернативы

- **Склейка в один namespace.** Отвергнуто: ломает пакеты, относительные
  импорты, `__name__`, провоцирует коллизии имён.
- **zipapp + base85, запись на диск, `sys.path`.** Отвергнуто: запись в общий
  управляемый каталог, замусоривание (движок чистит только `<plugin_id>.py`),
  медленнее, хрупко при обновлениях.
- **Рантайм-загрузка из соседней папки без сборки.** Отвергнуто как основной
  механизм: работает только при ADB-разработке, не годится для распространения.
  Частично переиспользуется в режиме `catalib watch --deploy`.

## Последствия

- Один валидный самодостаточный файл, совместимый со штатной установкой и
  распространением.
- Нормальные `import` между модулями плагина без изменения кода.
- Корректные трейсбеки на исходные файлы.
- Запасной путь при ограничениях Chaquopy: прямая регистрация в `sys.modules`
  через `exec` встроенных исходников в порядке зависимостей.
