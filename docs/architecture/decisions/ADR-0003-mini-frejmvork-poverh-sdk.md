# ADR-0003: Мини-фреймворк поверх SDK exteraGram

- **Статус:** принято
- **Дата:** 2026-05-17

## Контекст

Пользователь выбрал вариант «свой мини-фреймворк»: catalib задаёт структуру и
тонкий слой поверх SDK, а не только собирает файлы. Документированные типичные
ошибки разработки (незарегистрированные хуки, динамические метаданные, UI вне
UI-потока, отсутствие заглушек SDK для тестов) — повторяющийся шаблонный код,
который слой обязан устранить, не скрывая возможностей SDK.

## Решение

`catalib.support` предоставляет:

- Базовый класс плагина с единой точкой входа и автоматической регистрацией
  объявленных хуков и пунктов меню (через декларативные примитивы), что
  исключает класс ошибок «хук определён, но не зарегистрирован».
- Декларативные примитивы: декоратор хука, описание пункта меню, типизированные
  настройки поверх SDK-настроек.
- `support.sdk` — централизованные безопасные импорты SDK, обёрнутые в
  `try/except` с заглушками, что даёт офлайн-тестирование доменной логики без
  Android и единственную точку адаптации при обновлении SDK.
- Утилиты UI-потока и очередей с безопасными значениями по умолчанию.

Слой остаётся «тонким»: прямой доступ к объектам SDK не запрещается, фреймворк
лишь убирает шаблон и опасные умолчания. Метаданные плагина задаются в
`catalib.toml` и подставляются bundler-ом как литералы (требование AST-парсера
exteraGram), исключая ошибку «динамические метаданные».

## Альтернативы

- **Только сборка, прозрачно к SDK.** Отвергнуто пользователем; не устраняет
  повторяющийся шаблон и класс ошибок регистрации хуков.
- **Толстая абстракция, полностью скрывающая SDK.** Отвергнуто: ломается при
  обновлениях SDK, ограничивает доступ к возможностям, большой объём поддержки.

## Последствия

- Меньше шаблонного кода и ошибок в плагинах; единая точка адаптации к SDK.
- Появляется публичный API `catalib.support`, требующий стабильности и
  документирования и версионирования по SemVer.
- Граница сред (ADR-0002): `support` исполняется внутри плагина и не зависит от
  среды инструмента; проверяется тестом границы.
- На устройстве пакет `catalib` не установлен, поэтому модули
  `catalib.support` (и минимальный `catalib`) вендорятся в собранный файл и
  отдаются встроенным загрузчиком под их настоящими именами (модуль
  `bundler/vendor.py`).
- API SDK для пунктов меню фиксирован эмпирически (проверено на устройстве):
  `MenuItemData(menu_type=MenuItemType.{DRAWER_MENU|MESSAGE_CONTEXT_MENU|
  CHAT_ACTION_MENU|PROFILE_ACTION_MENU}, text=..., on_click=...,
  [icon=..., subtext=...])`, обработчик принимает `context: dict`.
  Декоратор `menu_item` и `support.sdk`-заглушки приведены к этому API.
