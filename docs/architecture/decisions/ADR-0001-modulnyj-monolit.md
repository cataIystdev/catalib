# ADR-0001: Модульный монолит вместо микросервисов

- **Статус:** принято
- **Дата:** 2026-05-17

## Контекст

Профессиональный рабочий процесс предписывает микросервисную архитектуру по
умолчанию. catalib — это offline CLI-инструмент сборки и встраиваемый рантайм-код.
У него нет сетевых границ, БД, очередей, независимо развёртываемых компонентов и
нагрузочных требований. Пользователь явно подтвердил поставку в виде одного
pip-пакета с CLI.

## Решение

Реализуется модульный монолит — один pip-пакет `catalib` со строго послойной
внутренней структурой. Каждый слой (`cli`, `manifest`, `bundler`, `runtime`,
`deploy`, `scaffold`, `support`) — отдельный подпакет с единственной
ответственностью и явными границами, готовый к независимому развитию.

## Альтернативы

- **Микросервисы.** Отвергнуто: накладные расходы не оправданы, нет распределённых
  взаимодействий, инструмент исполняется локально и однопроцессно.
- **Единый плоский модуль.** Отвергнуто: нарушает правило модуляризации,
  не масштабируется по сложности.

## Последствия

- Простая установка и запуск, минимум зависимостей.
- Дисциплина границ между слоями обеспечивается структурой пакетов и тестами,
  а не сетевыми контрактами.
- При необходимости любой слой может быть вынесен в отдельный пакет без
  переписывания вызывающего кода.
