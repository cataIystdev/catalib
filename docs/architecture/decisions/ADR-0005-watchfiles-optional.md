# ADR-0005: watchfiles — опциональная зависимость

- **Статус:** принято (частично ревизовано ADR-0011)
- **Дата:** 2026-05-18

> Ревизия (ADR-0011, 2026-05-19): отсутствие `watchfiles` больше **не**
> приводит к ошибке команды `watch` — есть stdlib-поллинг-фолбэк.
> Пакет остаётся опциональным и предпочтительным; раздел «Решение» ниже
> читать с этой поправкой.

## Контекст

`catalib watch` использует `watchfiles` для слежения за исходниками. Пакет
`watchfiles` содержит расширение на Rust и поставляется бинарными wheel'ами
под десктопные платформы. На телефоне (Termux, Pydroid) подходящего wheel
обычно нет, а сборка из исходников требует тулчейна Rust — практически
невозможна для конечного пользователя.

При этом `catalib` ценен на телефоне и без слежения: команды `build`, `init`,
`version` собирают и инициализируют плагин и не нуждаются в `watchfiles`.
Раньше `watchfiles` был жёсткой зависимостью, а `catalib.cli.app` импортировал
`watch_command`, который импортировал `watchfiles` на верхнем уровне. Поэтому
без `watchfiles` падал весь CLI, включая `build` и `version`.

## Решение

`watchfiles` вынесен из обязательных `dependencies` в опциональную группу
`watch` (`pip install "catalib[watch]"`). В группу `dev` он добавлен явно,
чтобы тесты и разработка сохранили слежение.

Импорт `watchfiles` отложен в тело команды `watch`: вспомогательная функция
`catalib.cli.watch_command._load_watch()` импортирует `watchfiles.watch`
лениво и при `ImportError` выдаёт понятную ошибку с подсказкой по установке,
завершая команду кодом 1. `catalib.cli.app` по-прежнему импортирует
`watch_command`, но это больше не тянет `watchfiles`, поэтому `build`/`init`/
`version` работают без него.

## Альтернативы

- **Заменить `watchfiles` на стандартный polling.** Отвергнуто: заметно хуже
  по отзывчивости и нагрузке; усложняет проверенный код слежения ради
  сценария, который и так покрыт ленивым импортом.
- **Динамически выбирать backend (`watchfiles` или polling).** Отвергнуто как
  избыточное на текущем объёме: дополнительная ветка и тесты без явного
  запроса; ленивый импорт + понятная ошибка решают задачу установки на
  телефоне минимальным изменением.
- **Оставить `watchfiles` обязательным, документировать ограничение.**
  Отвергнуто: не решает реальную проблему установки на телефоне.

## Последствия

- На телефоне: `pip install catalib` ставится без Rust-зависимости; доступны
  `build`/`init`/`version`. `watch` требует `pip install "catalib[watch]"`.
- Обратная совместимость сохранена: на десктопе с группой `watch`/`dev`
  поведение `watch` не изменилось.
- Инвариант (импорт `watchfiles` не нужен для импорта CLI и команды
  `version`) закреплён тестом в отдельном интерпретаторе.
