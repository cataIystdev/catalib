# ADR-0009: Типизация catalib и поставка SDK-стабов

- **Статус:** принято
- **Дата:** 2026-05-18

## Контекст

`catalib.support` уже аннотирован, но без маркера `py.typed` type-чекеры
(Pyright/Pylance/mypy) игнорируют аннотации установленного пакета. Кроме
того, плагин обращается к самим модулям SDK exteraGram (`base_plugin`,
`client_utils`, `file_utils`, `android_utils`, `hook_utils`, `ui.*`,
`extera_utils.*`), которые на машине разработчика **не импортируются**
(они есть только внутри приложения). Без стабов IDE не подсказывает их
API — главный пробел комфорта (см. анализ APK).

## Решение

1. Добавить маркер PEP 561 `catalib/py.typed` и классификатор
   `Typing :: Typed`; существующие аннотации `catalib.support` становятся
   видимыми type-чекерам без изменения кода.
2. Поставлять `.pyi`-стабы публичного SDK внутри пакета
   (`catalib/sdk_stubs/…`), сигнатуры — по официальной поверхности SDK
   (сверено с MCP-справочниками и декомпиляцией движка).
3. Команда `catalib stubs [--dir typings] [--force]` копирует стабы в
   каталог `typings/` проекта. Pyright/Pylance подхватывают `typings/`
   автоматически; для mypy — `mypy_path`.

Стабы именно **устанавливаются в проект**, а не остаются импортируемыми
модулями `catalib`, потому что они описывают сторонние модули
(`base_plugin` и т. д.), которых нет в окружении; стаб-пакеты type-чекеры
ищут по имени целевого модуля, а каталог `typings/` — стандартный путь.

## Альтернативы

- **Только `py.typed`, без SDK-стабов.** Отвергнуто: не закрывает главный
  пробел — автодополнение по SDK.
- **Отдельный пакет `catalib-sdk-stubs` на PyPI.** Отложено: лишняя
  публикация и синхронизация версий; поставка внутри catalib + команда
  установки проще и согласована по версии.
- **Инлайн-стабы как часть `catalib.support`.** Не подходит: стабы должны
  отвечать за имена сторонних модулей, а не за `catalib.support`.

## Последствия

- Полное автодополнение и проверка типов в IDE для `catalib.support` и
  для прямого доступа к SDK.
- `catalib stubs` — новый публичный контракт CLI; стабы версионируются
  вместе с пакетом, обновляются повторным запуском с `--force`.
- Стабы и `py.typed` входят в wheel/sdist (проверено сборкой).
