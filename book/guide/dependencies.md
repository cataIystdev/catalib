# Зависимости

Плагин может объявлять внешние pip-зависимости. exteraGram установит их при
первой загрузке плагина у пользователя.

## Объявление в манифесте

```toml
[plugin]
requirements = ["tinydb", "mpmath>=1.3"]
```

## Объявление в модуле

catalib также собирает `__requirements__` из модулей дерева и сливает с
манифестом (без дубликатов, в порядке первого появления):

```python
# src/storage/db.py
__requirements__ = ["tinydb"]
```

Итог — объединённое множество требований манифеста и всех модулей.

## Только pure-Python

exteraGram скачивает **только** «универсальные» wheels
(`*-none-any.whl`). Пакеты с бинарными расширениями не работают. catalib
отклоняет на этапе сборки явный блок-лист:

```
numpy, pandas, scipy, cryptography,
opencv-python, opencv-python-headless, opencv-contrib-python
```

Пример ошибки:

```
RequirementsError: 'numpy' (манифест): бинарный пакет 'numpy' не
поддерживается exteraGram (только pure-Python wheels)
```

Имя дистрибутива нормализуется по PEP 503, поэтому `opencv_python>=4.5`
тоже будет отклонён.

## Предустановленные библиотеки

Внутри exteraGram доступны без объявления (примерный список, зависит от
сборки): `requests`, `beautifulsoup4`, `lxml`, `pillow`, `PyYAML`,
`packaging`, `debugpy`. Их **не нужно** добавлять в `requirements`.

## Что пройдёт валидацию

Спецификаторы версий и extras сохраняются как есть:

```toml
requirements = [
  "requests[socks]>=2,<3",
  "tinydb ; python_version > '3'",
]
```

Пустые строки в списке — ошибка сборки.

## Практические советы

- Минимизируйте зависимости: каждая зависимость — это сетевая установка у
  пользователя при первом запуске.
- Предпочитайте стандартную библиотеку: `json`, `hashlib`, `base64`,
  `uuid`, `secrets`, `datetime`, `re`, `tomllib` доступны всегда.
- Если зависимость нужна только для офлайн-тестов инструмента, не кладите
  её в `requirements` плагина — держите в dev-зависимостях проекта.
