# catalib build

Собирает модульный плагин в один самодостаточный файл.

```bash
catalib build [--project DIR] [--check]
```

## Опции

| Опция | По умолчанию | Назначение |
|-------|--------------|------------|
| `--project`, `-p` | `.` (текущий каталог) | каталог проекта плагина (где лежит `catalib.toml`) |
| `--check` | выкл. | только проверить сборку, файлы не записывать |

## Что делает

1. Загружает и валидирует `catalib.toml`.
2. Обходит дерево `src` (пакеты, относительные имена, защита от traversal).
3. Сливает `__requirements__` манифеста и модулей, проверяет блок-лист.
4. Компилирует в один файл: литералы метаданных + встроенный загрузчик +
   таблица исходников + вендоренный `catalib.support`.
5. Проверяет результат: синтаксис и AST-валидность метаданных.
6. Записывает **два идентичных файла** в каталог `out`:
   `<plugin_id>.py` и `<plugin_id>.plugin`.

## Вывод

```
$ catalib build --project hello
Собран плагин 'hello': 4 модулей
Файлы: hello/dist/hello.py, hello/dist/hello.plugin
```

С зависимостями:

```
Собран плагин 'toolbox': 44 модулей
Зависимости: tinydb, mpmath
Файлы: example/dist/toolbox.py, example/dist/toolbox.plugin
```

## Режим проверки

```bash
catalib build --check
```

```
Собран плагин 'hello': 4 модулей
Проверка пройдена (файлы не записаны, --check).
```

Удобно для CI: убедиться, что плагин собирается и метаданные корректны,
ничего не записывая. Код возврата `1` — если сборка невозможна.

## Зачем `.py` и `.plugin`

Содержимое идентично побайтно. `.py` удобен при разработке и совместим с
инструментами; `.plugin` — расширение для установки в exteraGram. Какой
файл отдавать пользователю — зависит от способа установки в вашей сборке
exteraGram; оба валидны.

## Типичные ошибки

| Сообщение | Причина |
|-----------|---------|
| `Ошибка сборки: не найден catalib.toml в ...` | неверный `--project` |
| `Ошибка сборки: каталоги с модулями без __init__.py: core` | в пакете нет `__init__.py` |
| `Ошибка сборки: модуль точки входа 'plugin' не найден` | нет `src/plugin.py` или неверный `entry` |
| `Ошибка сборки: ... бинарный пакет ...` | бинарная зависимость в `requirements` |

Полный разбор причин — [Подводные камни](../troubleshooting.md).
