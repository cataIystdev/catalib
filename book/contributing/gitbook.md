# Настройка GitBook

Эта документация написана как GitBook-книга и синхронизируется из
GitHub-репозитория.

## Структура в репозитории

```
.gitbook.yaml          # конфиг GitBook (в корне репозитория)
book/
├── README.md           # вводная страница
├── SUMMARY.md          # оглавление (дерево страниц)
├── getting-started/ ...
├── guide/ ...
└── ...
```

`.gitbook.yaml`:

```yaml
root: ./book/

structure:
  readme: README.md
  summary: SUMMARY.md
```

- `root` — каталог с контентом книги (отделён от внутренней `docs/`
  с ADR/планами, чтобы не смешивать пользовательскую и проектную
  документацию).
- `structure.readme` — стартовая страница.
- `structure.summary` — файл оглавления.

## Файл `SUMMARY.md`

Оглавление и дерево страниц — вложенный markdown-список ссылок:

```markdown
# Содержание

- [Введение](README.md)

## Начало работы

- [Установка](getting-started/installation.md)
- [Быстрый старт](getting-started/quickstart.md)
```

Заголовки `##` становятся группами в боковой панели. Порядок ссылок —
порядок страниц. Каждая страница книги должна быть в `SUMMARY.md`.

## Подключение к GitBook.com (Git Sync)

1. Создайте организацию/пространство на [gitbook.com](https://gitbook.com).
2. В пространстве: **Sync → GitHub**, авторизуйте GitBook App для
   репозитория `cataIystdev/catalib`.
3. Выберите репозиторий и ветку (`main`), направление синхронизации
   (обычно двунаправленное GitHub ↔ GitBook).
4. GitBook прочитает `.gitbook.yaml`, возьмёт контент из `book/` и
   построит сайт по `SUMMARY.md`.
5. Каждый push в `main` обновляет опубликованную книгу; правки в редакторе
   GitBook возвращаются коммитами в репозиторий.

Никаких дополнительных файлов, кроме `.gitbook.yaml` и `book/`, не нужно.

## Локальный предпросмотр (Honkit)

`gitbook-cli` не поддерживается. Для локальной сборки удобно
**Honkit** (поддерживаемый форк GitBook), он понимает тот же
`SUMMARY.md`:

```bash
npx honkit serve book      # локальный сервер с автоперезагрузкой
npx honkit build book _site  # статическая сборка в _site/
```

Honkit — только для предпросмотра/статического экспорта; основная
публикация — через GitBook.com Git Sync.

## Правила для страниц

- Каждая страница — отдельный `.md` внутри `book/`, путь относительный.
- Ссылки между страницами — относительные пути (`../guide/hooks.md`).
- Новую страницу обязательно добавлять в `SUMMARY.md`, иначе GitBook её
  не покажет.
- Front-matter `--- description: ... ---` в начале файла — опционален
  (используется на стартовой странице).

## Поддержание актуальности

Документация — часть единицы работы. При изменении публичного поведения
(CLI, API `catalib.support`, формат манифеста) обновляйте
соответствующие страницы `book/` в том же коммите и добавляйте запись в
`CHANGELOG.md`.
