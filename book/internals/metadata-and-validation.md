# Метаданные и валидация

## Как exteraGram читает метаданные

Движок разбирает файл плагина через `ast` и берёт значения дандеров
**только если это строковые литералы** (`ast.Constant`). Динамические
значения (`__id__ = get_id()`, `__id__ = NAME + "x"`, f-строки) движок
игнорирует — плагин не загрузится корректно.

Поэтому метаданные задаются в `catalib.toml`, а компилятор эмитит их
строковыми литералами. В исходниках плагина писать дандеры не нужно и не
следует.

## Какие дандеры эмитятся

Из манифеста компилятор формирует блок:

```python
__id__ = "my_plugin"
__name__ = "My Plugin"
__version__ = "1.0.0"
__description__ = "..."        # если задано
__author__ = "..."             # если задано
__icon__ = "exteraPlugins/1"   # если задано
__app_version__ = ">=12.5.1"   # из min_version, если задано
__sdk_version__ = ">=1.4.4"    # из sdk_version, если задано
__requirements__ = ["tinydb"]  # если непусто
```

Канонический дандер версии приложения — **`__app_version__`**. catalib
**намеренно не эмитит** `__min_version__`: exteraGram показывает на него
отдельный экран несовместимости даже при совместимой версии приложения.
Это исправление конкретного бага (см. [Подводные камни](../troubleshooting.md)).

## Статическая проверка catalib

Перед записью компилятор проверяет (модуль `catalib.manifest.metadata`):

- результат **синтаксически валиден** (компилируется);
- каждый дандер — **строковый литерал** (а `__requirements__` — список
  строковых литералов);
- присутствуют обязательные `__id__`, `__name__`, `__version__`;
- `__id__` совпадает с именем выходного файла (`<plugin_id>`);
- `__version__` соответствует `N[.N[.N[.N]]]`.

Нарушение — `MetadataError`/`CompilerError`, файл не записывается. Это
ловит ошибки до устройства.

## Правила формата (модель манифеста)

| Поле | Паттерн |
|------|---------|
| `id` | `^[a-z][a-z0-9_]{1,31}$` |
| `version` | `^\d+(\.\d+){0,3}$` |
| `min_version` / `sdk_version` | `^(>=|<=|==|>|<)?\d+(\.\d+){0,3}$` |

`id` обязан быть валидным именем Python-модуля, потому что exteraGram
импортирует плагин как `importlib.import_module("<plugin_id>")`.

## Совместимость версий на устройстве

exteraGram сравнивает `__app_version__`/`__sdk_version__` с версией
приложения и SDK на устройстве. `__sdk_version__` — жёсткий гейт: если
устройство имеет меньшую версию SDK, установка блокируется с сообщением
вида `Plugin requires sdk version >=1.4.4, but current is 1.4.3.3`.
Поэтому `sdk_version` стоит задавать только при реальной необходимости.
